Note technique — Templates d’ordonnance (mPDF) — SOS Prescription

Ce document décrit les capacités et limitations pratiques du moteur PDF actuel (mPDF) afin d’itérer rapidement sur un template HTML/CSS sans modifier le PHP.

Principe
  • Le PDF est généré à partir d’un template HTML chargé depuis un fichier (source de vérité).
  • Le plugin remplace des tokens {{TOKEN}} par les données métier, puis mPDF rend le HTML en PDF.
  • Objectif : rendu stable, premium, et maintenable.

Outil de debug (Back-office)

  • Dans Back-office → SosPrescription → Ordonnance, activez l’option “Afficher les bordures de debug”.
  • Effet : injection d’un CSS de debug (td, th, div { border: 0.1mm solid red !important; }) lors de la génération mPDF.
  • Objectif : repérer visuellement les cellules/blocs qui “poussent” le layout (barcode, compteur médicaments, etc.).

1) Ce qui marche très bien avec mPDF

  • Tables (recommandées) : <table>, <tr>, <td> pour les mises en page “grille”.
  • Styles simples : font-family, font-size, font-weight, color, background, border, padding, margin.
  • Arrondis : border-radius (souvent OK, mais à garder raisonnable).
  • Images : <img> en data URI (base64) ou fichiers locaux (selon configuration).
  • Code-barres natif via balise mPDF <barcode> (ex: C128A).

2) Limitations / À éviter

  • CSS Grid / Flexbox avancé : support incomplet, rendu instable selon les cas.
  • Variables CSS (--var) : support partiel → préférer des valeurs fixes.
  • Max-height / max-width : pas toujours fiable → préférer height/width explicites.
  • JS dans le template : non exécuté côté mPDF (le HTML doit être “final” après remplacement des tokens).

3) Recommandations de “Premium” compatible

  • Construire la page avec 3 tables : En-tête, Corps (prescriptions), Footer (sécurité/signature).
  • Utiliser des badges (fond très clair + bord fin + texte bleu) plutôt que des layouts complexes.
  • Limiter les effets visuels ; privilégier la typographie et les espaces.

4) Tokens disponibles (extraits)

Le moteur remplace notamment :

  • {{DOC_NAME}}, {{DOC_TITLE}}, {{DOC_SPECIALTY}}, {{DOC_RPPS}}, {{DOC_PHONE}}
  • {{DOC_DIPLOMA_LINE}} (ex: “Diplômé Faculté Paris XIII • Lauréat de l’Académie”)
  • {{PATIENT_NAME}}, {{PATIENT_BIRTHDATE}}, {{PATIENT_AGE}}
  • {{PATIENT_WEIGHT}}, {{PATIENT_HEIGHT}} (affiche “—” si non renseigné)
  • {{RX_UID}}, {{RX_PUBLIC_ID}}, {{VERIFY_URL}}
  • {{DELIVERY_CODE}}, {{RX_FINGERPRINT_SHORT}}
  • {{MEDICATION_ROWS_HTML}} (liste HTML des lignes médicaments)
  • {{SIGNATURE_IMG_HTML}} (balise <img> prête à afficher)

5) Tester un nouveau template

  1. Aller dans Backoffice → SOS Prescription → Ordonnance.
  2. Uploader le fichier HTML → il devient le template actif (override).
  3. Générer une ordonnance depuis la console médecin et comparer au rendu attendu.
  4. En cas de problème, cliquer “Restaurer le template par défaut”.

Note : cette page est volontairement concise. Pour un rendu “pixel-perfect” proche navigateur, une solution headless (Chromium) est généralement nécessaire.
