Template “Vérification Pharmacien”

Page publique consultée lors du scan QR : /v/{token}

Objectif

Le template contrôle le rendu “premium” de la page de vérification sans toucher au PHP. Il est chargé depuis un fichier HTML (override uploadable depuis l’admin).

Source de vérité

Le fichier HTML est la source de vérité UI. Le serveur remplace des placeholders de type {{TOKEN}} et injecte certains blocs HTML (liste médicaments, actions…) pour garder une logique métier unique.

Placeholders supportés

Dans l’admin, une table affiche la liste des placeholders supportés et lesquels sont détectés dans votre template (présent/absent). Les placeholders inconnus sont signalés.

Blocs HTML injectés

  • {{MED_LIST_HTML}} : liste des médicaments (HTML déjà formatté par le serveur)
  • {{META_ROWS_HTML}} : métadonnées “Sécurité” (URL, identifiant, empreinte, code délivrance…)
  • {{PDF_ACTIONS_HTML}} : actions PDF (télécharger/voir)
  • {{DISPENSE_SECTION_HTML}} : statut + action “Marquer comme délivrée” (avec code 6 chiffres)
  • {{FLASH_HTML}} : messages succès/erreur après action

Conseil : ne modifiez pas la structure interne de ces blocs ; stylisez-les via CSS (classes).

Contraintes sécurité / confidentialité

  • La page peut être consultée sans compte (pharmacien). Pas d’informations inutiles : uniquement ce qui est requis pour vérifier.
  • Le token est un identifiant aléatoire long (non dérivé de l’UID). Évitez de le réafficher en clair partout ; privilégiez l’empreinte courte {{HASH_SHORT}}.
  • Le bouton “Marquer comme délivrée” doit demander le code à 6 chiffres imprimé sur l’ordonnance.

CSS recommandé

Cette page est du HTML “navigateur”. Vous pouvez utiliser flexbox, grid, variables CSS, badges… Néanmoins, pour garder une compatibilité maximale (thèmes, mobiles), privilégiez :

  • Une colonne centrée (max-width 1000px)
  • Des cartes (Card) avec bordure légère + ombre douce
  • Des boutons standards (primary/secondary) + états disabled

Exemple minimal

<div class="card">
  <div class="title">Vérification ordonnance</div>
  <div class="meta">
    {{META_ROWS_HTML}}
  </div>
  <div class="meds">
    {{MED_LIST_HTML}}
  </div>
  <div class="actions">
    {{PDF_ACTIONS_HTML}}
    {{DISPENSE_SECTION_HTML}}
  </div>
</div>

Protocole de test

  • Utiliser Prévisualiser (données de démo) depuis l’admin pour tester rapidement.
  • Vérifier : affichage patient, badge délivrance, téléchargement PDF, action “délivrée” (code 6 chiffres).
  • En cas d’erreur : relever le ReqID s’il est affiché et consulter l’onglet Logs.
